# Super Mario Web

Браузерная игра в стиле классического Mario, созданная в качестве выпускного проекта. Реализована на чистом JavaScript с использованием Canvas, Web Audio API и AJAX. Полностью адаптирована под мобильные устройства.

## Как запустить

1. Откройте страницу GitHub Pages по ссылке выше, либо просто запустите `index.html` в любом современном браузере.
2. На компьютере управляйте с помощью клавиш **стрелки** (влево/вправо) и **пробел/вверх** (прыжок).  
3. На телефоне используйте появившиеся виртуальные кнопки внизу экрана или сенсорные жесты (свайп влево/вправо для движения, тап вверх для прыжка).

## Соответствие критериям оценки

Ниже описано, как проект удовлетворяет каждому из критериев для получения оценки «отлично».

### 1. Плавная анимация объектов большой площади, в т.ч. на мобильном устройстве

Весь игровой процесс работает в цикле `requestAnimationFrame`, обеспечивая стабильные 60 кадров в секунду. Анимируются крупные фоновые слои (облака, холмы) через класс `ParallaxBackground`, персонажи, монеты и блоки также имеют покадровую анимацию. Плавность проверена на реальных смартфонах (Android, iOS).

### 2. Мультимедийные возможности JavaScript

- **Оригинальные графические построения:** все объекты (Марио с комбинезоном и буквой «M», враги-гумбы, кирпичные блоки, блоки с «?», финишный флаг, монеты) отрисованы примитивами Canvas без использования внешних изображений.  
- **Звуковые возможности:** с помощью Web Audio API генерируются звуки прыжка, сбора монеты, уничтожения врага, победы, а также фоновая мелодия во время игры (класс `SoundManager`).  
- **Вибрация:** при сборе монет и убийстве врагов вызывается `navigator.vibrate()` (работает на поддерживающих устройствах).

### 3. Обработка любого жеста на тачскрине

Класс `GestureHandler` обрабатывает `touchstart`, `touchmove`, `touchend`. Реализована поддержка:
- движения персонажа при скольжении пальца влево/вправо;
- прыжка при свайпе вверх;
- прыжка при обычном тапе.  
Дополнительно для удобства на мобильных экранах отображаются виртуальные кнопки (стрелки и кнопка прыжка), также работающие через тач-события.

### 4. AJAX и прогресс загрузки объёмных данных

- **AJAX:** после завершения игры счёт отправляется на сервер через `fetch` (метод `saveScoreAjax`). В демонстрационных целях используется `https://jsonplaceholder.typicode.com/posts`, но URL можно заменить на реальный серверный эндпоинт.  
- **Прогресс-бар загрузки:** при запуске приложения имитируется загрузка большого файла (изображение 5000×5000 пикселей) через `XMLHttpRequest`. Событие `onprogress` обновляет полосу прогресса и проценты. Предусмотрен таймаут 8 секунд, чтобы игра запустилась даже при проблемах с сетью.

### 5. Корректная работа кнопок браузера «вперёд», «назад», «обновить» и предупреждение о потере данных

- **History API:** при переходе между состояниями (`menu`, `playing`, `gameover`, `win`) выполняется `pushState` с соответствующими URL (`/game`, `/gameover`, `/win`). Обработчик `popstate` переключает экраны согласно истории, позволяя использовать кнопки «Назад» и «Вперёд».  
- **Предупреждение о потере данных:** во время активной игры срабатывает обработчик `beforeunload`, показывающий стандартное диалоговое окно при попытке обновить страницу или закрыть вкладку.

### 6. Полная адаптивность без горизонтальной прокрутки

- Canvas занимает 100% ширины и высоты окна браузера, размеры пересчитываются в методе `resize()` при изменении размеров окна.  
- Все элементы интерфейса используют относительные единицы измерения и функцию `clamp()`.  
- Для `body` задано `overflow: hidden`, а для canvas — `touch-action: manipulation`. Горизонтальная полоса прокрутки не возникает ни при каких разрешениях экрана.

### 7. Объектно-ориентированный подход

Код организован с использованием классов ES6:
- `SoundManager` — управление звуком;
- `Particle` — частицы для визуальных эффектов;
- `GameObject`, `Player`, `Enemy`, `Coin`, `QuestionBlock`, `Flag` — игровые объекты;
- `ParallaxBackground` — анимированный фон;
- `MarioGame` — главный контроллер игры;
- `GestureHandler` — обработка жестов.

Классы инкапсулируют данные и поведение, взаимодействуют через методы. Применены композиция и наследование (например, `Player` и `Enemy` расширяют `GameObject`).

### 8. Полная работоспособность на мобильном устройстве

Приложение протестировано на реальных Android- и iOS-устройствах:
- игра запускается и управляется касаниями/виртуальными кнопками;
- звук и вибрация работают (после первого взаимодействия для AudioContext);
- интерфейс адаптируется под маленькие экраны без горизонтальной прокрутки;
- все игровые механики (сбор монет, уничтожение врагов, активация блоков, финиш) полностью функциональны.
